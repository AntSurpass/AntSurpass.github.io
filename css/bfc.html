# bfc

(块级格式化上下文)属于文档流中的普通流，它是用来决定内部子元素如何布局，以及对其他元素的影响

触发bfc有以下值：

  • 根元素
  • float不为none
  • position为 absolute 或 fixed
  • overflow不为visible
  • column-count或 column-width 不为auto
  • diaplay 为 inline-block, tabel-cell,table-caption,flex,inline-flex,grid,inline-grid
   <div class="box">
    <img src="./css/1.jpg">
    <p class="bfc">
    最美不是下雨天，而是和你躲过的屋檐,最美不是下雨天，而是和你躲过的屋檐,最美不是下雨天，而是和你躲过的屋檐,最美不是下雨天，而是和你躲过的屋檐,最美不是下雨天，而是和你躲过的屋檐,最美不是下雨天，而是和你躲过的屋檐
    </p>
    </div>

    .box {
      width: 300px;
    }
   img {
     width: 100px;
     height: 100px;
     float: left;
   }
   .bfc {
     /* float:left */
     /* overflow: hidden; */
     /* display: flex; */
     /* display: grid */
     /* column-count: 30px; */
     /* column-width: 12px; */
   }

bfc 应用

  • 清除子元素浮动引起的父元素高度坍塌即清除浮动元素带来的影响
  • 解决两个元素margin重叠

# 边距重叠

垂直方向上发生的margin重叠现象，

折叠边距的值为正正取大，正负相加，负负取小

触发重叠场景:

  • 相邻的兄弟元素
  • 父元素与其第一个或最后一个子元素之间,子元素的外边距会“溢出”到父元素的外面。
  • 空的块级元素

注意

对于第二项，可能要复杂些，首先父元素没有触发bfc，其次如果是第一个元素，其父元素不存在padding-top，border-top，与父元素之间也没有行内元素。 如果是最后一个元素，其父元素不存在padding-bottom，border-bottom，与父元素之间也没有行内元素,并且父元素没有height，min-height,max-height。

<div class="m">
      <!-- <span>ss</span> 父元素之间有内联元素-->
      <div class="m-c"></div>
      <div class="m-c"></div>
</div>

   .m {
     width: 150px;
     background-color: red;
     /* overflow: hidden 触发bfc */
     border-top: 10px; // 父元素有border
   }
   .m-c {
     width: 50px;
     margin: 20px;
     background-color: purple;
   }

注意

对于空元素没有边框、内边距、行内元素、height、min-height
