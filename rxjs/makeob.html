    # 创建 observable

    of 发出一个observable数据流

    import {of} from 'rxjs'
const source$ = of(1, 2, 3)

    from 可以将可遍历的对象（iterable）转化为一个 Observable，from 还可以根据 promise 创建一个 Observable。我们用 fetch 或者 axios 等类库发送的请求都是一个 promise 对象，我们可以使用 from 将其处理为一个 Observable 对象。

    import {from} from 'rxjs'
const source$ = from([1, 2, 3])

    fromEvent 用 DOM 事件创建 Observable，第一个参数为 DOM 对象，第二个参数为事件名称。

    import { fromEvent } from 'rxjs';

const eleBtn = document.querySelector('#btn')
const click$ = fromEvent(eleBtn, 'click')

    fromEventPattern 将添加事件处理器、删除事件处理器的 API 转化为 Observable

    function addClickHandler (handler) {  document.addEventListener('click', handler)
} 
function removeClickHandler (handler) {  document.removeEventListener('click',handler)
}
 
fromEventPattern(
  addClickHandler,
  removeClickHandler
).subscribe(x => console.log(x))

    interval 操作符创造了一个数据流，interval(1000) 会产生一个每隔 1000 ms 就发出一个从 0 开始递增的数据

    import { interval } from 'rxjs';
import { map } from 'rxjs/operators'
    const source$ = interval(1000).pipe(
  map(x => x * x)
)

source$.subscribe(x => console.log(x))

    timer 则可以接收两个参数，第一个参数为发出第一个值需要等待的时间，第二个参数为之后的间隔时间。第一个参数可以是数字，也可以是一个 Date 对象，第二个参数可省。

    import { timer } from 'rxjs';

timer(5000,1000).subscribe(x => {
  console.log(x)
})

    range 操作符 of 产生较少的数据时可以直接写如 of(1, 2, 3)，但是如果是 100 个呢？这时我们可以使用 range 操作符

    import { range } from 'rxjs';

range(1, 100).subscribe(x => {
console.log(x) // 产生1到100的数
})

    defer 创建的 Observable 只有在订阅时才会去创建我们真正想要操作的 Observable。defer 延迟了创建 Observable，而又有一个 Observable 方便我们去订阅，这样也就推迟了占用资源

    import { defer } from 'rxjs';

defer(() => console.log("ss")).subscribe()// 这里订阅了 输出“ss"
defer(() => console.log("ssw")) // 这里没订阅没有输出
